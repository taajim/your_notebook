The short version

What this notebook is for

We started writing these posts mostly for myself — a way to make sure we actually understood a topic from my Operating Systems course rather than just recognizing it well enough to pass a quiz. Explaining deadlock avoidance or paging to an imaginary reader turned out to be a far better test of understanding than re-reading the textbook one more time.

The project showcase grew out of the same habit. Most of what's here started as a coursework assignment or a "let me just try this" weekend build. None of it is polished beyond what a small side project needs to be — but each one taught me something I didn't know going in, and that's really the point of keeping a notebook like this in the first place.

This site itself is one of those projects: a static, responsive blog and showcase built with hand-written HTML and CSS, with no framework doing the layout work for me. Building the design system from scratch — colors, type scale, components — taught me more about CSS specificity than any tutorial had.
